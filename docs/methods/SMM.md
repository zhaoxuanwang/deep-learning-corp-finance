# Simulated Method of Moments

I estimate unknown parameters of the risky debt model in @strebulaev2012 [section 3.6] using Simulated Method of Moments. The detailed implementation procedures are summarized as below:

## SMM Implementation

### Notation

| Symbol | Definition |
|---|---|
| $\beta^*$ | True structural parameters underlying the real-world data. Unknown. |
| $\beta$ | A candidate parameter vector evaluated during optimization. |
| $\hat{\beta}$ | The SMM estimate: the $\beta$ that minimizes $Q(\beta)$. If the model is correctly specified, $\hat{\beta} \xrightarrow{p} \beta^*$. |
| $x$ | Real-world dataset (one panel, e.g., Compustat), generated by the unknown $\beta^*$. |
| $K$ | Number of parameters to estimate. |
| $R$ | Number of moments ($R \geq K$). |
| $S$ | Number of independently simulated panels per evaluation of $Q$. |
| $M(x)$ | $R \times 1$ moment vector from the real data panel. |
| $m_s(\beta)$ | $R \times 1$ moment vector from simulated panel $s$, for $s = 1, \ldots, S$. |
| $\bar{m}(\beta)$ | $R \times 1$ simulated moment vector averaged across $S$ panels: $\bar{m} = \frac{1}{S}\sum_{s} m_s$. |
| $W$ | $R \times R$ positive definite weighting matrix. |

### Algorithm summary

1. **Setup.** Compute data moments $M(x)$. Draw and fix simulation shocks $U$.
2. **Estimation Loop.** Minimize $Q(\beta)$ over $\beta$. The optimizer iterates over many candidates, each time solving the model at that $\beta$, simulating data, and computing $Q$. Convergence is determined by the optimizer's internal criteria.
3. **Optimal weight (Efficient Estimator).** Run Part 2 with $W = I$ to get $\hat{\beta}_1$. Construct $\hat{\Omega}$ at $\hat{\beta}_1$. Set $W = \hat{\Omega}^{-1}$. Run Part 2 again to get $\hat{\beta}$.
4. **Inference.** At $\hat{\beta}$, compute standard errors and the overidentification test.

In practice, the main bottleneck is the estimation loop in step 2: the minimization optimizer typically loop over large number of candidate $\beta$ values, and each iteration requires re-solving the model via VFI/PFI or deep learning methods. Standard optimizer such as `scipy.optimize` are designed to find the global optimum of $\beta^*$ but it can take hundrends or thousands of iterations and is computationally expensive.

---

### Part 1: Setup

Done once before estimation begins.

**Data moments.** Choose $R \geq K$ moments. Compute them from the real data $x$:

$$M(x) = \begin{bmatrix} M_1(x) \\ \vdots \\ M_R(x) \end{bmatrix}$$

These are fixed target numbers computed from $x$ alone, without knowledge of $\beta^*$.

**Simulation shocks.** Draw uniform random shocks $U$ for $S$ panels once. These are reused at every evaluation of $Q$ (common random numbers). Different candidates $\beta$ transform the same $U$ into different simulated data through the policy functions, while the underlying randomness is held constant. This makes $Q(\beta)$ smooth in $\beta$, reducing the number of optimizer iterations needed.

For example, for models with AR(1) process:
$\ln z′=\rho \ln z+ \sigma \varepsilon$ and $\varepsilon \sim N(0, \sigma_\varepsilon^2)$. The generic SMM solver still draw random uniform $u∼U(0,1)$ and then let the specific model convert it to $\varepsilon = \Phi^{-1}(u)$ with 
the standard normal inverse CDF $\Phi^{-1}$.

In practice, I fix a simulation seed and regenerate shocks deterministically at each evaluation. This produces identical shocks across all $β$ candidates without storing the full tensor.

---

### Part 2: Estimation

This part minimizes $Q(\beta)$ over $\beta$. It consists of an optimization loop that repeatedly evaluates $Q$ at different candidates until convergence.

#### Evaluate $Q(\beta)$

A function that takes a single candidate $\beta$ and returns the scalar $Q(\beta)$. Everything below is conditional on the given $\beta$.

1. **Solve the model** at $\beta$ (Bellman iteration, bond pricing equilibrium, etc.) to obtain the optimal policy functions for this $\beta$.
2. **Simulate** $S$ panels $\{\tilde{x}_1(\beta), \ldots, \tilde{x}_S(\beta)\}$ using these policy functions and the fixed $U$.
3. **Compute moments** from each panel $s$, yielding $R \times 1$ vectors $m_s(\beta)$ for $s = 1, \ldots, S$. Average across panels:

$$\bar{m}(\beta) = \frac{1}{S}\sum_{s=1}^S m_s(\beta)$$

4. **Form the $R \times 1$ error vector.** Choose one form and use it consistently throughout all parts:

$$e(\beta) = \bar{m}(\beta) - M(x) \qquad \text{(level)}$$

$$\text{or} \qquad e_r(\beta) = \frac{\bar{m}_r(\beta) - M_r(x)}{M_r(x)} \qquad \text{(percent)}$$

Level deviation is the **default** (`error_type="level"`) and is recommended whenever moments span a wide range of magnitudes: percent normalization would amplify moments whose target value is small (e.g., $\Pr(\text{default})\sim 10^{-4}$) by $1/M_r^2$, inflating the Omega_hat condition number by orders of magnitude and breaking the efficient estimator. Percent deviation can still be requested via `error_type="percent"` for settings where moments are naturally comparable in magnitude. When percent is selected and $M_r \approx 0$, the denominator is stabilized to $\text{sign}(M_r) \times \max(|M_r|, 10^{-6})$.

5. **Return** the objective value:

$$Q(\beta) = e(\beta)^\top \, W \, e(\beta)$$

Also retain the $S$ individual per-panel moment vectors $\{m_s(\beta)\}_{s=1}^S$ for constructing $\hat{\Omega}$ if needed.

#### Optimization loop

Starting from an initial guess $\beta^{(0)}$, a numerical optimizer calls `Evaluate` at successive candidates and converges to the minimizer:

$$\hat{\beta} = \arg\min_\beta \; Q(\beta)$$

Each evaluation requires a full model solve, so the number of evaluations dominates wall time. The choice of optimizer critically affects this count and thus practical feasibility. See the "Optimization" section below for the detailed design.

---

### Part 3: Optimal weight (two-step)

**First step.** Set $W = I_R$. Run Part 2 to obtain $\hat{\beta}_1$.

**Construct $\hat{\Omega}$.** At $\hat{\beta}_1$, retrieve the $S$ per-panel moment vectors $\{m_s(\hat{\beta}_1)\}_{s=1}^S$ saved by `Evaluate`. For each panel $s$, form the $R \times 1$ per-panel error vector using the same form (level or percent) as in Part 2:

$$E_s = m_s(\hat{\beta}_1) - M(x) \qquad \text{(level, $R \times 1$)}$$

$$\text{or} \qquad (E_s)_r = \frac{m_{s,r}(\hat{\beta}_1) - M_r(x)}{M_r(x)} \qquad \text{(percent, with same stabilization as Part 2)}$$

Estimate the $R \times R$ variance-covariance matrix of per-panel moment errors:

$$\hat{\Omega} = \frac{1}{S}\sum_{s=1}^{S} E_s \, E_s^\top$$

**Second step.** Set $W = \hat{\Omega}^{-1}$ and warm start from $\hat \beta_1$. Run the local refinement optimizer to find $\hat \beta_2 = \arg\min_\beta Q(\beta)$ using the updated $W$. This step is fast because $\hat \beta_1$ is already consistent. The goal here is to improve efficiency by updating the weight matrix.

$S > R$ is necessary for $\hat{\Omega}$ to be full rank. Use $S \gg R$ (e.g., $S = 200$ for $R = 13$) for a stable estimate.

**More refinement steps (optional).** We could further recompute $\hat{\Omega}$ at the latest $\hat{\beta}$, update $W$, re-run Part 2. Stop when $\|W_{i+1} - W_i\| < \epsilon$. I did not implement this in practice because the two-step is usually sufficient.

---

### Part 4: Inference

This part is done once at the final $\hat{\beta_2}$.

Following @hennessy2007costly, I compute the two test statistics: t-test statistics from standard errors for each $\hat \beta_k$, and the over-identifying test $J$-stat.  

**Jacobian.** Define the $R \times K$ Jacobian of the error vector:

$$D_{rk} = \frac{\partial e_r(\beta)}{\partial \beta_k}\bigg|_{\beta = \hat{\beta}} \approx \frac{e_r(\hat{\beta} + h_k \mathbf{e}_k) - e_r(\hat{\beta} - h_k \mathbf{e}_k)}{2 h_k}$$

where $\mathbf{e}_k$ is the $k$-th unit vector and $h_k = \max(10^{-4} |\hat{\beta}_k|, \; 10^{-8})$. Each evaluation of $e_r$ requires a full `Evaluate` call. The full Jacobian requires $2K$ evaluations.

**Standard errors.** The error vector $e = \bar{m} - M$ has variance $\text{Var}(e) = \Omega/S + \Omega = (1 + 1/S)\,\Omega$, where $\Omega/S$ is simulation noise (average of $S$ panels) and $\Omega$ is target noise (one data panel). By the delta method at the optimum with $W = \hat{\Omega}^{-1}$:

$$V = \left(1 + \frac{1}{S}\right) \left(D^\top W D\right)^{-1}$$

$$\text{se}(\hat{\beta}_k) = \sqrt{V_{kk}}$$

As $S \to \infty$, the simulation noise vanishes and $V \to (D^\top W D)^{-1}$.

The t-statistics for testing $H_0: \hat \beta_k = \beta^0_k$ are constructed as: 

$$t_k \equiv \frac{\hat \beta_k - \beta^0_k}{\text{se}(\hat{\beta}_k)} $$

Reject $H_0$​ at significance level $\alpha$ if $|t_k| > z_{1-\alpha/2}$.

@hennessy2007costly [Appendix B] explains the intuition: local identification implies the Jacobian determinant to be non-zero, meaning that the moments defined in $e_r$ are informative about $\beta$. If not, then $\partial e_r(\beta)/\partial \beta \approx 0$ will cause the variance $V_{kk}\to\infty$.

**Overidentification test.** Requires $R > K$ and $W = \hat{\Omega}^{-1}$. With $K$ parameters and $R$ moments, there are $R - K$ overidentifying restrictions beyond the minimum needed for identification. We aim to test the null that the simulated moment matches the real moment at the true $\beta^*$:
$$
H_0: \exist \beta^* \quad \text{such that} \quad e(\beta^*)=\bar{m}(\beta^*)-M(\beta^*)=0
$$
Under $H_0$, the scaled quadratic form at $\hat{\beta}$ is asymptotically chi-squared:

$$J = \frac{S}{S + 1} \; Q(\hat{\beta}) \;\xrightarrow{d}\; \chi^2(R - K)$$

The factor $S/(S+1)$ corrects for the target being one random draw (not a population quantity); as $S \to \infty$ and $\hat \beta \to \beta^*$, it converges to 1 and $J \to Q(\beta^*)=0$.

For significance level $\alpha$, reject $H_0$ if $J > \chi^2_{1-\alpha}(R-K)$. A rejection means no $\beta$ can match all $R$ moments simultaneously and indicates model misspecification. Importantly, correct solution means we should *fail to reject* the null (i.e., our model is mis-specified), but it cannot prove that our model is correctly specified.

### Pre-Estimation Diagnostics

Before running the full optimization, I compute three diagnostics at the known true parameters $\beta_0$ to verify that the estimation problem is well-posed. These checks are independent of the optimizer and isolate pipeline correctness from identification strength.

#### Diagnostic 1: Oracle Test

**Definition.** Evaluate the objective at the true parameters:

$$Q(\beta_0) = e(\beta_0)^\top W \, e(\beta_0), \qquad e(\beta_0) = \bar{m}(\beta_0) - M(\beta_0)$$

where $\bar{m}(\beta_0)$ is computed from $S$ simulated panels using a *different* random seed than the target $M(\beta_0)$, so the two are statistically independent. Report both $Q$ and $\|e\| = \sqrt{Q}$.

**Intention.** This tests whether the solver-simulation-moment pipeline produces correct moments at the known truth. It is a **necessary condition** for SMM to work: if $Q(\beta_0)$ is large, then the pipeline has a bug (wrong sign, off-by-one in panel indexing, inconsistent moment definition, etc.) and no optimizer can succeed. It is **not sufficient**: $Q(\beta_0) \approx 0$ confirms pipeline correctness but says nothing about whether $\beta_0$ is a unique minimizer or whether the optimizer can find it.

**Interpretation.** Pass if $Q < 0.01$ (equivalently $\|e\| < 0.1$). A small but nonzero $Q$ reflects simulation noise from using independent seeds; this vanishes as $S \to \infty$. A large $Q$ indicates a bug. Common failure modes: moment function uses wrong variable, simulation horizon too short for burn-in, solver did not converge.


#### Diagnostic 2: Jacobian Rank and Sensitivity

**Definition.** Compute the $R \times K$ Jacobian of the moment function at $\beta_0$ via central finite differences:

$$D_{rk} = \frac{\partial \bar{m}_r(\beta)}{\partial \beta_k}\bigg|_{\beta_0} \approx \frac{\bar{m}_r(\beta_0 + h_k \mathbf{e}_k) - \bar{m}_r(\beta_0 - h_k \mathbf{e}_k)}{2 h_k}$$

where $h_k = 0.01 \times (\beta_k^{\max} - \beta_k^{\min})$ scales the step to 1% of the feasible range per parameter. This requires $2K$ model solves. From $D$, compute:

**Singular values.** The SVD $D = U \Sigma V^\top$ yields singular values $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_K \geq 0$. Each $\sigma_j$ measures the moment sensitivity along the $j$-th principal parameter direction (a linear combination of the original parameters defined by the columns of $V$). Directions with large $\sigma_j$ are well-identified; directions with $\sigma_j \approx 0$ are unidentifiable.

**Rank.** $\text{rank}(D) = \#\{j : \sigma_j > 10^{-8}\}$. Full rank ($= K$) is required for local identification.

**Column norms.** For each parameter $k$, the column norm $\|D_{\cdot k}\| = \sqrt{\sum_r D_{rk}^2}$ measures the total moment sensitivity to that individual parameter. Classify as:

| $\|D_{\cdot k}\|$ | Status | Meaning |
|---|---|---|
| $< 10^{-6}$ | DEAD | No moment responds to this parameter |
| $< 0.05$ | WEAK | Moments respond but very faintly |
| $\geq 0.05$ | OK | Adequate sensitivity |

**Full Jacobian table.** The matrix $D$ itself (rows = moments, columns = parameters) shows exactly which moments identify which parameters. Entry $D_{rk}$ is the marginal effect of parameter $k$ on moment $r$. A column of near-zeros indicates an unidentified parameter. Two columns with proportional entries indicate confounded parameters — the moments cannot distinguish one from the other.

**Intention.** The rank condition ($\text{rank}(D) = K$) is the standard **necessary condition for local identification** (Rothenberg, 1971): it ensures the mapping $\beta \mapsto m(\beta)$ is locally injective at $\beta_0$, so the system $m(\beta) = m_{\text{target}}$ has a locally unique solution. The singular values quantify how strongly each direction is identified. The column norms and full Jacobian table provide interpretable per-parameter and per-moment diagnostics.

The rank condition is **not sufficient** for global identification (multiple well-separated minima may exist) or for good finite-sample performance (a direction with $\sigma_j = 0.01$ is technically identified but requires very precise optimization and large $S$ to pin down).

**Interpretation.**

- $\text{rank}(D) < K$: at least one parameter direction is locally unidentifiable. Either add moments that load on the missing direction, or calibrate the unidentified parameter externally.
- $\text{rank}(D) = K$ but $\sigma_K \ll \sigma_1$: all parameters are locally identified, but the condition number $\kappa(D) = \sigma_1/\sigma_K$ measures the sensitivity ratio between the best- and worst-identified directions. Large $\kappa$ (e.g., $> 10^3$) means the optimizer must navigate a narrow valley — feasible but requiring more function evaluations.
- WEAK or DEAD parameters: fix them externally or add targeted moments. Estimating a DEAD parameter wastes optimizer budget and inflates standard errors for all other parameters (the flat direction creates ridges in the objective surface that the optimizer slides along).
- Two columns with similar patterns: the corresponding parameters are confounded. The optimizer can trade off one against the other without changing $Q$ much. Consider whether additional moments with orthogonal sensitivity can break the confounding.


#### Diagnostic 3: $\hat{\Omega}$ Condition Number

**Definition.** At the first-step estimate $\hat{\beta}_1$ (or at $\beta_0$ during pre-estimation diagnostics), retrieve the $S$ per-panel error vectors $\{E_s\}_{s=1}^S$ (as defined in Part 3) and compute:

$$\hat{\Omega} = \frac{1}{S}\sum_{s=1}^S E_s E_s^\top \qquad (R \times R)$$

Report $\kappa(\hat{\Omega}) = \sigma_{\max}(\hat{\Omega}) / \sigma_{\min}(\hat{\Omega})$.

**Intention.** This checks whether $W = \hat{\Omega}^{-1}$ is numerically stable. A well-conditioned $\hat{\Omega}$ is **required for the efficient estimator** (Stage 2 with optimal $W$), the $J$-test, and the efficient standard errors from Part 4. It is **not an identification condition**: a poorly conditioned $\hat{\Omega}$ does not mean the parameters are unidentified. The first-step estimator ($W = I$) remains consistent regardless of $\hat{\Omega}$ — it will converge to $\beta_0$ as optimizer budget and sample size grow, but with larger variance than the efficient estimator.

**Interpretation.**

- $\kappa < 10^6$: $\hat{\Omega}^{-1}$ is numerically stable. Proceed with Stage 2 using $W = \hat{\Omega}^{-1}$. The $J$-test and efficient SEs are available.
- $\kappa > 10^6$: $\hat{\Omega}^{-1}$ is unreliable. Fall back to $W = I$ for Stage 2. The $J$-test and efficient SEs are not available. The first-step estimator is still consistent but not efficient.
- Common cause: $S$ is too small relative to $R$. The sample covariance $\hat{\Omega}$ is estimated from $S$ observations of an $R$-dimensional vector; when $S/R$ is small, sampling noise inflates the condition number. Rule of thumb: $S \gg R$ (e.g., $S \geq 50R$). Moments with very different scales (e.g., default frequency $\sim 10^{-4}$ vs leverage $\sim 1$) also increase $\kappa$ even in the population.


#### Summary: What Each Diagnostic Can and Cannot Conclude

| Diagnostic | Checks | Condition type | Failure consequence |
|---|---|---|---|
| Oracle $Q(\beta_0) \approx 0$ | Pipeline correctness | Necessary for SMM | Bug in solver/simulation/moments |
| $\text{rank}(D) = K$ | Local identification | Necessary for unique $\hat{\beta}$ | Unidentified parameter direction |
| SVs of $D$ | Identification strength | Quantitative (not binary) | Weak directions → slow convergence, large SEs |
| $\kappa(\hat{\Omega}) < 10^6$ | Efficient estimator feasibility | Required for $J$-test and efficient SEs only | Fall back to $W = I$; lose $J$-test |

The diagnostics are ordered by severity: an oracle failure blocks everything; a rank-deficient Jacobian means estimation is futile for those parameters; a large $\kappa(\hat{\Omega})$ only prevents the efficient second step but does not invalidate the first-step estimator.

## SMM Validation via Monte Carlo

### Purpose

Verify that the SMM implementation correctly recovers known parameters before applying it to real data. The idea is standard Monte Carlo simulation: 
- Simulate $J$ set of "real" data under known $\beta_0$, which is similar to the actual datasets like Compustat. 
- Then pretend that $\beta_0$ is unknown and verify whether our SMM method can recover $\hat \beta \approx \beta_0$ across $J$ simulations. Each simulation $j$ runs a full two-step SMM procedure and obtain a $\hat \beta^{(j)}$.

### Procedure

**Setup.** Pick and fix true parameters $\beta_0$.

**For each replication $j = 1, \ldots, J$:**

1. Simulate a "real" panel at $\beta_0$ with a fresh random seed. Compute target moments $M^{(j)}$.
2. Run the full two-step SMM (as described in the SMM recipe) using $M^{(j)}$ as the data targets. Use a **different** random seed for the simulation shocks as the "real" and simulated panels must be independent.
3. Store: estimate $\hat{\beta}^{(j)}$, standard errors $\text{se}^{(j)}$, and $J$-statistic.

Use $J \geq 10$ replications. Do a single-replication quick check ($J = 1$, verify $\hat{\beta} \approx \beta_0$) before committing to the full run.

### Evaluation Metrics

To measure the effectiveness and correctness of the solutions. I compute and report the following across $J$ replications per parameter $k$:

| Metric | Formula | Target | Interpretation |
|---|---|---|---|
| Bias | $\frac{1}{J}\sum_j \hat{\beta}_k^{(j)} - \beta_{0,k}$ | $\approx 0$ | One biased param implies its identifying moment is poorly computed or weak |
| SD | $\text{sd}(\{\hat{\beta}_k^{(j)}\})$ | Small | Measures overall estimator variability across replications |
| SE | $\frac{1}{J}\sum_j \text{se}_k^{(j)}$ | Small | Used for per-parameter t-tests |
| RMSE | $\sqrt{\frac{1}{J}\sum_j(\hat{\beta}_k^{(j)} - \beta_{0,k})^2}$ | Small | Overall accuracy (bias, variance, or both) |
| $J$-test size | Fraction where $J^{(j)} > \chi^2_{0.95}(R-K)$ | $\approx 0.05$ | $J \sim \chi^2(R−K)$ under true $H_0$ by construction |

**SD and SE:** Under sufficient optimizer budget and large $J$, SD $\approx$ SE. With limited training budget, optimizer noise may inflate SD beyond SE. The key diagnostic metrics are bias, t-test, and J-test. SD and RMSE should be reasonably small.

## Optimization

The main bottleneck of SMM is the optimization step $\hat \beta = \arg \min Q(\beta)$. Evaluating $N$ candidates of $\beta$ requires re-solving the dynamic model $N$ times. Most optimizers require hundreds or thousands of evaluations to converge, meaning that even if each model solve takes just 15 seconds, SMM estimation can take hours.

I use `scipy.optimize` rather than `tf.keras.optimizers` because the model solve (VFI, discrete default decisions) is not differentiable through standard automatic differentiation. SciPy supports bounds, global search, and computes gradients via finite differences when needed. The overhead of converting between NumPy and TF tensors is negligible relative to the model solve cost.

The optimization proceeds in two phases: (1) a **global search** to find the right basin, followed by (2) a **local refinement** to converge to $\hat{\beta}$ within that basin. I implement two modes that share the same global search but differ in the local refinement strategy.

### Global search: `dual_annealing`

Both modes use `scipy.optimize.dual_annealing` for Stage 1 global search. This is the simulated annealing approach used in @hennessy2007costly. It explores the bounded parameter space stochastically to avoid local minima, running for a fixed budget of `maxiter` iterations. After `dual_annealing` terminates, the local refinement optimizer is run from the best point found.

### Local search: derivative-free polish with Powell

The baseline mode uses `scipy.optimize.minimize` with `method="Powell"` for local refinement. Powell is a derivative-free conjugate-direction method: it treats $Q(\beta)$ as a black-box scalar and searches along coordinate directions without computing gradients.

| Phase | Method | Input | Output |
|-------|--------|-------|--------|
| Stage 1 global | `dual_annealing` | initial guess, bounds | coarse basin |
| Stage 1 polish | `Powell` | best point from above | $\hat{\beta}_1$ |
| Stage 2 | `Powell` | $\hat{\beta}_1$, updated $W$ | $\hat{\beta}_2$ |

Powell is robust and requires no gradient information, but its convergence is linear. It does not exploit the least-squares structure of the SMM objective.

## Application to the basic investment model

For this application, I follow @hennessy2007costly and fix the risk-free interest rate and depreciation at calibrated value: $r=0.04$ and $\delta=0.15$. This reduces the number of parameters to be identified. 

### Clean SMM Validation with frictionless benchmark
The basic investment model with no frictions (adjustment cost $\psi_0=\psi_1=0$) provide an ideal setting to validate the SMM implementation because it provides a clean separation of concerns:

- Optimal policy is directly provided in analytical closed-form and called by SMM. No VFI/deep learning methods inside the loop.
- Any errors and failures can only be in SMM rather than in model solution

I select 4 moments to match 3 target parameters so that I can conduct over-identification test. 

| # | Moment | Definition | Identifies | Reasons
|---|---|---|---|---|
| 1 | Mean investment / assets | $\mathbb{E}\bigl[I_{it}/k_{it} \bigr]$ |$\alpha$ | Higher $\alpha$ increases marginal product of capital |
| 2 | Var of investment / assets | $\text{Var}\bigl[I_{it} / k_{it}\bigr]$ |$\alpha, \sigma_\epsilon$ | How much investment responds to shocks. Higher $α$ amplifies the response; higher $σ$ increases shock dispersion | 
|3	| Serial corr of investment | $\text{Corr}(\frac{I_{it}}{k_{it}}, \frac{I_{i,t-1}}{k_{i,t-1}})$ |	$\rho$	| Higher $ρ$ means more persistent shocks, so consecutive investment decisions are more similar and serially correlated
| 4 | AR(1) variance | $\hat{\sigma}_u$ (residual std dev from same AR(1)) | $\sigma_\varepsilon$ | Direct approximation


### End-to-end application to frictional basic model

Once the SMM is validated, I apply it to the full basic investment model with capital adjustment costs $\psi>0$. This model no longer has analytical solution to the optimal policy $k'$, so we need to call a model solver inside the estimation loop. I use the Policy Function Iteration (PFI) and Euler Residual Minimization (ER) as two backend solver.

I select 5 moments to match 4 target parameters:

| # | Moment | Definition | Identifies | Reasons
|---|---|---|---|---|
| 1 | Mean investment / assets | $\mathbb{E}\bigl[I_{it}/k_{it} \bigr]$ |$\alpha$ | Marginal product of capital
| 2 | Var of investment / assets | $\text{Var}\bigl[I_{it} / k_{it}\bigr]$ |$\alpha, \psi_1$ | Higher adjust cost $\psi_1$ dampens investment responses (lower variance)
| 3 | Serial corr of investment | $\text{Corr}\!\left(\frac{I_{it}}{k_{it}},\; \frac{I_{i,t-1}}{k_{i,t-1}}\right)$ | $\psi_1, \rho$ | Partial adjustment lead to positive serial corr
| 4 | Serial corr of income / assets | $\hat{\beta}_1$ from panel AR(1) | $\rho, \psi_1$ | Direct approx for $\rho$, indirect measure of partial adjustment
| 5 | Std dev of shock to income | $\hat{\sigma}_u$ (residual std dev from same AR(1)) | $\sigma_\varepsilon$ | Direct approx

## Application to the risky debt model

### Definitions and Preparation

#### Equity issuance moments

To identify the equity issuance cost $\Omega(e)=(\eta_0 + \eta_1 |e|)\mathcal{I}\{e<0\}$, I follow @hennessy2007costly and use four cross-sectional moments that separate the fixed cost ($\eta_0$, which gates issuance frequency) from the proportional cost ($\eta_1$, which shapes the pecking order). The H&W set also drops the variance-of-issuance moment that they use for the quadratic cost $\lambda_2$, since this model has no quadratic term.

| # | Moment | Definition | Identifies | Reasons |
|---|---|---|---|---|
| 1 | Avg equity issuance / assets | $\mathbb{E}\bigl[\max(0, -e_{it}) / k_{it}\bigr]$ | $\eta_0$ | Overall issuance intensity; large $\eta_0$ → infrequent but large flotations → small mean |
| 2 | Frequency of equity issuance | $\Pr(e_{it} < 0)$ | $\eta_0$ | Direct identifier: large fixed cost gates how often the firm issues at all |
| 3 | Corr(equity issuance, investment) | $\text{Corr}\!\left(\frac{\max(0,-e_{it})}{k_{it}},\; \frac{I_{it}}{k_{it}}\right)$ | $\eta_0, \eta_1$ | Position of equity in the pecking order: strong correlation ⇒ low equity cost; weak correlation ⇒ firm prefers internal funds first. We use the correlation (scale-stabilised) rather than H&W's raw covariance because $\text{Cov}(\text{Iss}/k, I/k)$ has population variance $\sim 10^{-8}$ — both variables are near-zero for most firm-years — which would make Omega_hat singular regardless of sample size. Correlation is bounded in $[-1,1]$ and conditioning is comparable to other moments. |

**Moments computed but excluded from the active set** (kept in `_ALL_MOMENT_NAMES` for diagnostic comparison, disabled at the API level via `disabled_moments=[...]`):

- *Conditional issuance size* $\mathbb{E}[\max(0,-e)/k \mid e<0]$ — Jacobian audit shows zero loading on $\eta_1$ in this model; mechanically redundant with moments 1 and 2 since $\text{Cond Iss} = \text{Avg Iss}/\Pr(e<0)$.
- *Autocorr of equity issuance* — Jacobian shows $\eta_0$ loading is dominated by $\alpha$ and $\psi_1$; small target value ($\sim -0.01$) inflates Omega_hat condition number.
- *Cross-corr leverage → issuance* — Jacobian shows $\eta_1$ loading exists but is dominated by $\alpha$ and $\psi_1$; replaced by the cleaner Cov(Iss, I) pecking-order moment from H&W.

**Unused moment candidate** (defined in `_UNUSED_MOMENT_CANDIDATES`, not computed):

- *Frequency of negative debt* $\Pr(b' < 0)$ — H&W 2007 propose this to identify the precautionary-saving motive, but the production solver uses `adaptive=True` which shrinks the b-grid to the default-risk region ($b' > 0$ only). The moment is therefore structurally zero by construction; including it would create a singular row in Omega_hat.

#### Debt and leverage moments

Faithful to @hennessy2007costly: book net debt-to-assets identifies the level of $c_{\text{def}}$ (high default cost ⇒ lower equilibrium leverage), and the covariance between leverage and investment identifies the position of debt in the financing pecking order ($c_{\text{def}}$ controls how much firms substitute debt for internal funds when financing investment).

| # | Moment | Definition | Identifies | Reasons |
|---|---|---|---|---|

#### Investment and adjustment-cost moments

Mean investment / assets is H&W 2007's primary identifier for $\alpha$ (production curvature scales the equilibrium investment rate). The serial correlation of $I/k$ identifies $\psi_1$: higher convex costs force partial adjustment, creating positive autocorrelation that scales monotonically with $\psi_1$.

|---|---|---|---|---|

| 8 | Serial corr of investment | $\text{Corr}\!\left(\frac{I_{it}}{k_{it}},\; \frac{I_{i,t-1}}{k_{i,t-1}}\right)$ | $\psi_1$ | Partial adjustment |


#### Real technology moments

The variance of investment rate captures how much investment responds to shocks — firms respond less when $\alpha$ is small. The two AR(1) parameters are identified from a panel IV regression on the observable income/asset ratio $y_{it} = z_{it} k_{it}^{\alpha-1}$, following the same IV first-difference estimator used in the basic investment model (see above).

| # | Moment | Definition | Identifies | Reasons
|---|---|---|---|---|
| 9 | Mean investment / assets | $\mathbb{E}\bigl[I_{it}/k_{it}\bigr]$ | $\alpha$ | Equilibrium investment rate scales with $\alpha$ |
| 10 | AR(1) persistence | $\hat{\beta}_1$ from panel IV regression of $\Delta \log y_t$ on $\Delta \log y_{t-1}$ | $\rho$ | Direct |
| 11 | AR(1) shock std dev | $\hat{\sigma}_u$ from the same IV regression | $\sigma_\varepsilon$ | Direct |

#### Summary

- **Active moment set:** 11 moments — 3 issuance (H&W) + 2 leverage + 2 investment/adjustment + 3 real technology + 1 default frequency — to identify up to 7 parameters $(\alpha, \psi_1, \eta_0, \eta_1, c_{\text{def}}, \rho, \sigma_\varepsilon)$.
- **Diagnostic-only moments:** 4 additional moments are computed (`conditional_issuance_size`, `autocorr_equity_issuance`, `crosscorr_leverage_issuance`, `default_frequency` when $c_{\text{def}}$ is calibrated) and reported alongside the active set, but excluded from the SMM objective. These appear in the panel-moments output for comparison but never enter $Q(\beta)$.
- **API toggle:** Pass `disabled_moments=[...]` to `RiskyDebtEnv.make_smm_spec()` to exclude any moment from the active set without removing it from the registry. Validation ensures the remaining active set still satisfies $R \geq K$.
- **Overidentifying:** With all 7 parameters estimated and 11 active moments, $R - K = 4$ overidentifying restrictions allow for the $J$-test. When parameters are calibrated externally (e.g., $c_{\text{def}}$), moments tagged solely to those parameters are auto-dropped, preserving the overidentifying margin.
- **Cost:** Each candidate $\beta$ requires a full nested VFI solve to obtain the optimal policy $\pi(\cdot;\beta)$. Computing all 14 moments per panel adds negligible overhead relative to the solver.
